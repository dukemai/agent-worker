## Domain Terms

| Term | Definition |
|------|------------|
| **Bucket** | Time-based task grouping: `today`, `this_week`, `later`. Stored in separate tables (`today_tasks`, etc.) with FK to `tasks`. |
| **Renewal** | Expiration reminder (passport, subscription, membership). Stored as `tasks` with `metadata.item_type = "renewal"`. Dual-date: `expires_on`, `renew_by`. Escalation: T-30, T-14, T-7, T-1. |
| **Growing** | Stockholm seasonal gardening — seed, transplant, prune, harvest. Uses `growing_profiles`, `growing_windows`, `growing_suggestions_log`. Suggestions generated Sun/Wed; shown in digest Mon/Fri. |
| **Family context** | User preferences (shopping list, plant list, interests). Key-value in `family_context` table. |
| **Promo grocery watchlist** | Dashboard `/promo-grocery-watchlist`; `family_context.key = promo_watchlist` — JSON string array of grocery items for promo/flyer matching → [promo-watchlist.md](requirements/promo-watchlist.md). |
| **Promo meal plan (AI)** | Gemini returns **10 meal suggestions** from a chosen `promo_match_*` import — [promo-meal-plan.md](requirements/promo-meal-plan.md). |
| **Promo meal suggestions** | Near-term: **10 meal ideas** from the same import (mockup + future API) — [promo-meal-suggestions.md](requirements/promo-meal-suggestions.md). |
| **Daily digest** | Morning email (06:30 Stockholm) — tasks, weather, renewals, promotions, learning. Growing section included Mon/Fri only. Generated by Worker cron, sent via Resend. |
| **Ingestion** | Email → Worker → postal-mime → Gemini → Supabase. Tasks extracted and bucketed. |
| **Learning profile** | Topic/category-based curriculum. Lessons generated by Gemini. Feedback loop (easy/hard, more/less). |
| **Shared shopping list / cook plan** | **Recipes** → **plan to cook** → **prepare** (at home vs need) → **shopping list** + share link — [shared-shopping-list.md](requirements/shared-shopping-list.md). |
| **Promo shopping pipeline (phases)** | Ordered sub-phases A–G: shared list → crawl → DB promos → matching → triggers → pantry/meals → integration — [promo-shopping-pipeline-phases.md](requirements/promo-shopping-pipeline-phases.md). |

## Schema Quick Ref

| Table | Purpose |
|-------|---------|
| `tasks` | All tasks and reminders (source, metadata, status) |
| `today_tasks` / `this_week_tasks` / `later_tasks` | Bucket membership (task_id FK) |
| `family_context` | Key-value preferences |
| `promo_match_runs` / `promo_match_items` | Dashboard import of `watchlist-matches-only.json`; items include `week_number` (ISO, UTC) |
| `cook_plans` / `cook_plan_items` | (Planned) **Plan to cook** — queued `saved_recipes` — [shared-shopping-list.md](requirements/shared-shopping-list.md) |
| `shared_shopping_lists` / `shared_shopping_list_items` | (Planned) Shareable buy lists after **prepare** — see [shared-shopping-list.md](requirements/shared-shopping-list.md) |
| `growing_profiles` | City, space type, experience, interests |
| `growing_sources` | YouTube/blog URLs, transcript, status |
| `growing_windows` | Seasonal activity catalog (Stockholm) |
| `growing_knowledge` | Reference nuggets from sources |
| `growing_suggestions_log` | Weekly suggestions + conversion to tasks |
| `learning_profile` | Curriculum settings per topic |
| `learning_log` | Lessons + feedback |

Full schema in [SPEC.md](SPEC.md).
